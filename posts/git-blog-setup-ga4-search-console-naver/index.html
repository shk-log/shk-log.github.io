Git 블로그(Hugo) GA4 통계 및 검색 노출 설정하기 (구글 서치콘솔, 네이버)

December 9, 2025 · 3 min · JSON

블로그를 처음 개설하고 방문자 통계를 보고 싶었지만, GitHub Pages 자체적으로는 통계 기능을 지원하지 않는다. 그래서 구글 애널리틱스(GA4)를 연동하고, 내친김에 구글과 네이버 검색 노출(SEO) 설정까지 진행했다.

Git 블로그 GA4 구글 애널리틱스 설정#

구글애널리틱스 홈페이지에 들어가 GA4 설정을 진행했다. 설정 가이드가 있지만 난 해봤던 관계로 그냥 진행했다. 처음이시라면 구글 애널리틱스 공식 가이드를 참고하시길 바란다. 이 글을 쓰기 위해 공식 문서를 다시 봤는데, 확실히 개발 배경지식(API 등)이 생기니 예전보다 훨씬 수월하게 읽혔다.. 아마 Git 블로그를 작성하시는 분들은 개발지식이 어느정도 있는분이고 금방 따라하실수 있을거다.

소유권 인증#

구글 애널리틱스 설정을 위해선 이 블로그가 내 블로그인지 소유권 인증을 할 필요가 있다. 이 인증을 위해서 예전 부터 흔히 하던것이 애널리틱스에서 제공하는 코드를 html head 태그 안에 넣는 것이다. 가입후 현재 내 깃블로그는 Hugo를 사용해 만들었고, pageMod 테마를 사용하고 있다. 이 환경에서는 hugo.yaml 에 내 ga4 아이디를 넣으면 된다.

services:
	googleAnalytics:
		id: G-XXXXXX  # 여기에 내 아이디 입력
# 들여쓰기 주의

내 아이디는 ‘G-’ 로 시작한다. 소유권인증을 위해 삽입하라는 코드를 자세히 들여다보면 금세 찾을 수 있다. 이거 입력하고 소유권 인증하면 끝이다.

내부 트래픽 제한#

통계가 시작되지만 정확한 내용을 보기 위해선 내부 트래픽을 제외할 필요가 있다.

내부 트래픽 제한하는 곳#

관리 > 속성설정 > 데이터 수집 및 수정 > 데이터 스트림 > '내 블로그 선택' > Google 태그 > 태그 설정 구성 > 설정 자세히보기 > 내부 트래픽 정의

여기서 만들기 하고 ‘규칙 이름, traffic_type, ip주소’를 넣으면 된다. 나는 IP주소가 다음과 같음 을 누르고 현재 내 컴퓨터의 IP를 입력했다. 여러곳에서 작업하면 내부아이피가 여러개가 될 수 있다. 그경우 traffic_type을 기본값인 internal로 동일하게 하길 권한다.

데이터 필터 수정#

내 내부데이터를 데이터상에서 식별하려면 필터상태를 기본값인" 테스트" 로 두면 되고, 테스트를 통해 내부 아이피가 잘 걸러진다. 이제 데이터상에서 보고싶지 않다. 라고 하면 필터 상태를 “사용중"으로 두면 된다.

데이터 필터 수정하는 곳#

관리 > 속성설정 > 데이터 수집 및 수정 > 데이터 필터

구글 검색 노출 | 구글 서치 콘솔 Google Search Console#

이왕 한김에 구글 검색 노출도 진행하려고 했는데, 소유권 인증이 너무 간단하게 되버렷다. 애널리틱스가 되니 그냥 구글 서치콘솔도 소유권을 인증했다. 구글 만세!

구글 서치 콘솔 사이트 바로가기 사이트 맵 제출하는 곳(색인 생성 > sitemaps)이 있는데, 이걸 해야 구글 봇이 원활히 내 글을 가져갈 수 있다. Git 블로그의 경우 블로그 메인 URL 끝에 /sitemap.xml 입력후 이동하면xml 파일이 나오면. 새사이트 맵추가 뒤에 sitemap.xml 을 입력해서 제출하면 된다.

매번 글을 쓰고 색인요청 해야 하나?#

글을 쓰면 구글 봇이 알아서 내 블로그를 구글 검색에 노출 시킨다. 다만, 초반에 깃 블로그를 작성할 때는 구글 봇이 자주 방문 안할것이다. 대형사이트와 비교해도 현저히 적게 방문할 것이다. 그래서 글을 쓰면 바로바로 내 글 URL을 색인 요청 한다.

첫글 색인요청한지 하루가 넘었는데 아직URL 이 구글에 등록되어 있지 않다. 당연할거다. 색인요청을 해서 바로 색인되는게 아니라 색인 대기열에 올려놓는 거니까. 글을 10개정도 더써보고 문제가 있다면 다시 확인해보기로 하고 넘어간다.

네이버 검색 노출 | 네이버 서치 어드바이저#

네이버 검색 노출이 잘될지는 모르겠지만 일단 해본다. 안될거 같다는 생각으로 가능성을 재단할 필요는 없으니 말이다.

네이버 서치 어드바이저 Naver Search Advisor 바로가기

구글 애널리틱스는 Hugo 자체 설정(services)으로 간단히 해결됐지만, 네이버 서치 어드바이저는 Hugo나 테마 설정에 의존하기보다 ‘HTML 파일 업로드’ 방식을 선택했다. 서버 루트에 직접적으로 파일을 배치하는 방식은 테마와 무관하게 영구적으로 인증을 유지할수 있기 때문에 더 안정적인 방법이라고 여겼기 때문이다.

내 사용자 인증 파일을 다운받은후 내 블로그 페이지에 자동생성된 폴더중 하나인 static에 복사해 두면 끝이다. 마치 자바 스프링 부트 환경에서 Gradle이나 Maven으로 빌드하면 static폴더 있는게 루트 폴더로 가듯, Hugo 명령어(Hugo -t PagerMod)로 빌드를 하면 static 폴더에 있는 네이버 사용자 인증 파일이 루트폴더로 들어간다. 웹 개발의 공통적인 표준이 이런것인가 라는 생각을 해본다.

단 두개의 글이지만 Git블로그 하길 잘했단 생각이 든다. 부딪히다보니 git이 더 친숙하게 느껴진다. 오늘 글과 관련된 내용으로 유의미한 데이터 변화나 특이사항이 생기면 다시 공유하도록 하겠다.
